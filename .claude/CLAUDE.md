## 응답 스타일

다음 상황에서 항상 적용:

- **변경 제안 시**: AS-IS vs TO-BE 비교
- **여러 옵션 제시 시**: 각 옵션의 장단점
- **어떤 안을 추천할 때**: 그 안의 한계까지 함께 명시 — 단점을 가리지 말 것
- **새 개념 설명 시**: 다음 순서로 풀어낼 것
  1. 등장 전 문제 상황
  2. 해결 아이디어/철학
  3. 그래서 등장한 기술
  4. 구현 방식 — 핵심 동작원리 3개
  5. 장단점과 한계
